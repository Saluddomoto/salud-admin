# AIO(AI検索最適化)・SEO 対策リスト — salud-web.jp

> 作成日: 2026-07-08
> AIO = ChatGPT・Gemini・Google AI Overviews などの「AIの回答」に引用・推薦されるための最適化(LLMO/GEOとも)。
> 実装フェーズ: WordPress実装時に組み込む(デザインモックには含まれない)

## なぜ重要か

補助金を探す経営者は「ものづくり補助金 いくらもらえる」「補助金 申請代行 費用」を
**Googleではなく ChatGPT や AI Overviews に聞く**ことが急増している。
AIは「構造が明確で、数字と定義がはっきりした、信頼できる運営者のページ」を引用する。

## 対策チェックリスト

### 1. 構造化データ(JSON-LD) — AIがページの意味を理解するための機械可読データ
- [ ] `Organization` + `LocalBusiness`(株式会社Salud、経営革新等支援機関の認定情報)
- [ ] `Service`(補助金申請支援・融資支援・WEB制作・AI開発の各サービス)
- [ ] `FAQPage`(FAQページ + 各LPのQ&A部分)← **AIの回答に最も引用されやすい**
- [ ] `Article` + `author`(ブログ記事に専門家の監修者情報)
- [ ] `BreadcrumbList`(パンくず)
- [ ] `Offer` / `PriceSpecification`(料金: 基本料金10万〜+成功報酬10〜15%を機械可読に)

### 2. E-E-A-T(信頼性) — AIは「誰が言っているか」を重視
- [ ] 専門家の実名・資格・写真・プロフィールページ(住田守弘氏=中小企業診断士、栗原淳氏=行政書士 ほか8名)
- [ ] 記事に「監修: ◯◯(行政書士)」を表示 + 構造化データ連携
- [ ] 運営者情報・会社概要・認定支援機関番号を明記
- [ ] 一次情報の発信(自社の採択実績・相談データ→AIは一次情報を優遇)

### 3. AIクローラー対応(技術)
- [ ] robots.txt で AI クローラーを許可(GPTBot / ClaudeBot / Google-Extended / PerplexityBot)
- [ ] `llms.txt` を設置(サイト概要・主要ページをAI向けに要約したファイル)
- [ ] WordPressのSSR(サーバー側レンダリング)なので JS 依存問題なし ✓
- [ ] ページ速度・モバイル対応(Core Web Vitals)

### 4. コンテンツ構造 — AIが引用しやすい書き方
- [ ] 各ページ冒頭に「結論・定義文」(例:「ものづくり補助金とは、〜に最大4,000万円が交付される制度です」)
- [ ] 見出し(h2/h3)を質問形にする(「ものづくり補助金はいくらもらえる?」)
- [ ] 数字・期限・料金は**表**で明示(AIは表を最優先で抽出する)
- [ ] 比較コンテンツ(制度比較表・自社と他社の違い)
- [ ] ブログは「定義→金額→対象→流れ→FAQ」のテンプレで統一

### 5. すでにデザインに織り込み済みの AIO 有利要素 ✓
- FAQ ページ・各LPのQ&A(→ FAQPage スキーマを載せるだけ)
- 料金の明示(基本料金+成功報酬を隠さない=AIが「料金が明確な会社」として引用しやすい)
- 専門家実名の体制ページ
- 制度ごとの個別LP(1ページ1テーマ=AIが引用しやすい単位)
- 締切・上限額の数値データ

## 実装タイミング
WordPress実装の際にテーマへ組み込む(JSON-LDはテンプレートで自動出力)。
