# 株式会社Salud AI開発プロジェクト

本プロジェクトは、**株式会社Salud**が提供する各事業セグメントにおいて、最先端 of AI技術を統合・活用することにより、「売上向上」「業務効率化」「競合との差別化」を最大化し、事業の非連続な成長を実現するためのAI開発プロジェクトです。

---

## 1. プロジェクトの背景と目的

株式会社Saludは、補助金申請支援、Web/LP制作、SEO・Webマーケティング、セミナー運営、パートナー（代理店）制度など、多角的なビジネスを展開しています。
本プロジェクトでは、それぞれの事業プロセスにおける属人化の解消、作業時間の劇的な削減、および提供価値の向上を目指し、各領域に特化したAIソリューションの企画・開発・導入を推進します。

---

## 2. 主要なAIシステムコンセプト

本プロジェクトで構想・開発を進める主要なAIシステムは以下の通りです。

### ① 社内共通基盤：AI連携型スマート進捗管理＆スケジュール共有ボード
*   **課題**: 複数事業の並行によるスタッフ間の進捗状況・スケジュールのブラックボックス化。
*   **AIソリューション**: チャット（Slack/LINE等）での報告をAIが解析して進捗ステータスを自動更新。また、締切からマイルストーンを逆算してスケジュールを自動生成し、タスクを割り振ります。

### ② 補助金申請支援：補助金申請ドラフト生成＆適合性判定AI
*   **課題**: 申請書類（事業計画書など）の作成にかかる専門知識と膨大な作成工数。
*   **AIソリューション**: 顧客のWebサイトやヒアリングシートから事業内容を抽出し、各種補助金への適合度判定、および申請書の一次ドラフトを自動生成します。

### ③ パートナー（代理店）制度：パートナー専用 AI営業アシスタント
*   **課題**: パートナー側の製品・補助金知識の差による営業停滞と、本部のサポート負荷。
*   **AIソリューション**: LINEやSlack上でパートナーからの質問に即座に回答し、顧客に合わせた提案スクリプトや提案資料の構成を自動生成します。

### ④ Web制作・LP制作：ペルソナ・訴求軸＆構成案ジェネレーター
*   **課題**: 構成案（ワイヤーフレーム）作成やセールスコピー構築にかかるディレクション工数。
*   **AIソリューション**: 商材とターゲット層から「ペルソナ」「強み」「訴求軸」を整理し、ワイヤーフレームおよびセールスコピーの一次構成案を瞬時に作成します。

### ⑤ SEO・Webマーケティング：SEO記事設計＆マーケティングレポート自動化
*   **課題**: キーワード調査・構成案作成の属人化、GA4/サチコ等を用いたレポート作成コスト。
*   **AIソリューション**: 上位競合サイトの構成を分析した見出し設計の自動化、およびアクセス解析データからの改善レポート自動生成。

### ⑥ セミナー運営：セミナー企画＆マルチユース・コンテンツ生成AI
*   **課題**: セミナー企画・集客文面の作成工数と、セミナー動画資産の未活用。
*   **AIソリューション**: セミナー動画からサマリー記事、Q&A、SNS用ハイライト動画の台本などをマルチユースで自動生成します。

## 3. 導入・開発ロードマップ（優先順位）

実務のボトルネック解消と投資対効果（ROI）を考慮し、以下のステップで順次実装を推進します。

| 優先順位 | 対象システム | 目的 | 期待効果 |
| :--- | :--- | :--- | :--- |
| **1位 (高)** | **⑥ 社内案件管理・スケジュール共有** | インフラ整備・ミス防止 | 全スタッフのタスク・進捗の可視化、報告コスト削減 |
| **2位 (高)** | **① 補助金申請ドラフト生成AI** | 業務効率化・売上向上 | 書類作成工数の劇的削減、同一リソースでの対応件数倍増 |
| **3位 (高)** | **⑤ パートナー専用 AI営業アシスタント** | 売上向上・競合差別化 | パートナー経由の売上レバレッジ、本部サポート工数削減 |
| **4位 (中)** | **② LP・Web構成案生成AI** | 業務効率化・競合差別化 | 受注率の向上、初期ディレクションの高速化 |
| **5位 (中)** | **③ SEO記事設計・レポート自動化** | 業務効率化 | 既存マーケティング運用の内製化コスト削減 |
| **6位 (低)** | **④ セミナー運営自動化AI** | 業務効率化 | セミナー動画のコンテンツ二次利用の高速化 |

## 4. プロジェクトの進め方

1.  **要件定義・設計**: 各フェーズ対象のシステム要件、利用ツール（Googleカレンダー、Slack等）との連携設計。
2.  **プロトタイプ開発**: 主要AIモジュールのAPI連携、簡易プロンプトエンジニアリングによる検証。
3.  **社内テスト・フィードバック**: 実際の業務データを用いた精度検証とUX調整。
4.  **本番導入・運用**: 業務プロセスへの統合と継続的なプロンプト・学習データの改善。
